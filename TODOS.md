## P1 — V1.5 Data Expansion (Post-Hackathon Polish)

**What:** Add 6 new data sources discovered during deep research. These use the same CKAN API patterns already in Agent 2 and require minimal build effort.

**New parameters to add:**
1. **Heritage Formerly Listed** (`heritage-formerly-listed`, ~3,700 records) — properties removed from Heritage Register due to Bill 23. 15 min build, same shapefile pattern as heritage register.
2. **Residential Fire Inspections** (`residential-fire-inspection-results`, 124,414 records, daily refresh) — building-level Ontario Fire Code compliance for high-rise residential. 30 min build. Replaces intersection-level `fire-incidents` as primary fire risk signal.
3. **Property Tax Relief Programs** — hardcoded rules by buyer profile (first-time, investor, downsizer). Surface eligible rebates, deferrals, and subclass reductions. 30 min build.
4. **Transit Dividend** (`ttc-routes-and-schedules`, GTFS) — distance to nearest subway/streetcar, car vs. TTC cost differential over 10 years. Already shown in PDF frontend. 1 hr build.
5. **Building Violations** (`building-construction-demolition-violations`, 48,035 records, daily refresh) — open compliance issues. 30 min build, same CKAN pattern.
6. **Basement Flooding Study Areas** (`basement-flooding-study-areas`, 67 subsewersheds) — sewer-related flood risk complement to TRCA. 30 min build.

**Why:** Expands Toronto Open Data usage from 6 to 12+ sources. Directly strengthens the judging criterion score. Each uses the same `httpx` + CKAN API pattern already built in Agent 2.

**Effort:** M (human ~3h total) / S (CC+gstack ~30min)
**Priority:** P1 — do immediately after hackathon core is stable
**Depends on:** V1 pipeline working end-to-end

## P2 — RAPIDS cuDF GPU Acceleration

**What:** Add RAPIDS cuDF for GPU-accelerated spatial radius queries on the 400K+ building permit records.

**Why:** CPU shapely + haversine on 400K records is functional but slow. cuDF on the GB10 Blackwell GPU is estimated 10-50x faster. A side-by-side demo ("CPU: 8.2s. GPU: 0.4s.") is a memorable NVIDIA hardware moment that upgrades the GB10 story from "inference box" to "compute platform."

**Context:** Deferred from V1 due to dependency setup risk under 24h timeline. `cudf` and `cuspatial` conda packages may conflict with the existing Python environment on GB10. The spatial query is in Agent 2's retrieval layer — the CKAN permit bbox + haversine queries at 500m (`sources/ckan.py::within_radius`).

**Effort:** M (human ~2h) / S (CC+gstack ~20min)
**Priority:** P2 — should land for V2 demos
**Depends on:** Core pipeline working and stable on GB10
