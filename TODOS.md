## P2 — RAPIDS cuDF GPU Acceleration

**What:** Add RAPIDS cuDF for GPU-accelerated spatial radius queries on the 400K building permit records.

**Why:** CPU GeoPandas on 400K records is functional but slow. cuDF on the GB10 Blackwell GPU is estimated 10-50x faster. A side-by-side demo ("CPU: 8.2s. GPU: 0.4s.") is a memorable NVIDIA hardware moment that upgrades the GB10 story from "inference box" to "compute platform." Directly addresses the NVIDIA hardware utilization judging criterion.

**Context:** Deferred from V1 due to dependency setup risk under 24h timeline. `cudf` and `cuspatial` conda packages may conflict with existing GeoPandas install on GB10. Once the core pipeline is stable post-hackathon, this is a 2-hour integration. The spatial query is in Agent 2's retrieval layer — the CKAN permit radius queries at 500m.

**Effort:** M (human ~2h) / S (CC+gstack ~20min)
**Priority:** P2 — should land for V2 demos
**Depends on:** Core pipeline working and stable on GB10

## P2 — Nemotron 120B Model Upgrade

**What:** Config switch from Llama 3.1 8B to Nemotron 120B MoE via NIM for Agents 1, 3, and 4.

**Why:** GB10 has 128GB unified memory — enough for Nemotron 120B MoE. This changes the pitch from "we're running 8B locally" to "we're running 120B parameters on edge hardware." Dramatically better synthesis quality, especially for the negotiation leverage framing in Agent 4. 30-minute config change, no architectural changes required.

**Context:** Decision checkpoint at Hour 1 of the hackathon build. If 8B NIM benchmark shows single-call latency <5s, the upgrade is viable — 3 sequential calls at 15s each is still under 30s total. If 8B is already slow (>10s/call), do NOT upgrade. The upgrade is a NIM model swap: change `model` parameter in all httpx calls to NIM.

**Effort:** S (human ~30min) / S (CC+gstack ~5min)
**Priority:** P2 — decision checkpoint at Hour 1
**Depends on:** NIM 8B latency benchmark at Hour 1
