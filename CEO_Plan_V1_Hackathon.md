## Vision

### 10x Check
The 10x version of Meridian isn't a better cost calculator — it's a real-time buyer's advocate that negotiates on your behalf. Imagine entering an address and getting not just hidden costs but a negotiation brief: "Heritage Part IV + 3 active structural permits gives you $40,000-60,000 in documented leverage. Here are the exact clauses to add to your offer." The agent doesn't just reveal risk — it turns risk into power. That's Meridian 1.5, and it's achievable with one additional prompt instruction in Agent 4.

### Platform Potential
The 4-agent architecture (Planning → Retrieval → Analysis → Synthesis) is intentionally extensible:
- V2: Swap Retrieval with Retrieval + Flinks → personal financial resilience layer
- V3: Add comparison engine → buy vs. rent decision, neighbourhood comparison
- B2B: License the risk engine to mortgage brokers as a pre-screening tool

Every agent boundary is a clean handoff via JSON contract, which means each agent can be upgraded independently without rebuilding the whole pipeline.

## Accepted Scope (added to this plan)

- **Address autocomplete + geocoding:** Toronto Address Points CKAN dataset serves BOTH autocomplete AND primary geocoding (returns lat/lon). **Nominatim is removed entirely** — not a fallback (public-endpoint policy bans demo use; Address Points miss → geocode_confidence=Unknown, spatial checks skipped). See PRD_Backend_Build.md §7.1.
- **List price input:** `st.number_input('List price ($)', min_value=100000, step=50000)` alongside address field. Required for LTT and mortgage calculations. 5-minute fix.
- **Agent 2 executes query plan:** Agent 2 reads Agent 1's `query_plan[]` and skips sources marked SKIP. RentSafeTO is only queried when Agent 1 identifies property as condo or high-rise residential. This makes the planning genuinely adaptive, not theatrical.
- **Negotiation leverage framing:** Agent 4 synthesis prompt addition: "For each RED flag, include one sentence of negotiation leverage with a dollar range from the cost breakdown. Use ranges, never point estimates."
- **Per-source confidence indicator:** Each data source result in the reasoning trace UI displays a colored badge: HIGH (data returned, recency known), MEDIUM (data returned, recency unknown), LOW (partial data), UNKNOWN (timeout or error).
- **Error rescues (5):** json.JSONDecodeError on CKAN, FileNotFoundError on local GeoJSON, EmptyLLMResponse, json.JSONDecodeError on LLM output, httpx.ConnectError on NIM.
- **UI edge cases (2):** Submit lock via session_state['analyzing'], explicit GREEN verdict text for zero-flag properties.

## Architecture (with accepted cherry-picks)

**Clarification:** Agent 2 is pure orchestration code (no LLM call). It is the async parallel fetch layer built in Phase 1 (Hours 2-5). It is NOT a separate LLM prompt.

**Property types Agent 1 classifies:** `detached_urban`, `detached_suburban`, `semi_detached`, `condo`, `condo_townhouse`, `commercial_mixed` (6 canonical types per PRD_Backend_Build.md §7.2). RentSafeTO is queried only for `condo` and `commercial_mixed`/high-rise residential.

**Confidence annotation rules:**
- HIGH = HTTP 200 + `last_updated` / `extract_date` field present in CKAN response
- MEDIUM = HTTP 200, no recency metadata
- LOW = HTTP 200, empty results array (0 records found)
- UNKNOWN = timeout (>5s), 4xx, or 5xx response

**Pre-cache schema:** Store demo property results as `data/demo_cache/<address_slug>.json` matching Agent 2 output schema. Retrieval layer checks `DEMO_MODE=true` env var to bypass live API calls and serve cached JSON.

**Agent boundary JSON contracts** (full schemas in `Office_Hours_Gap_Analysis_and_Build_Strategy.md`):
- Agent 1 → Agent 2: `{address, coordinates, property_type_inferred, query_plan[], reasoning_trace}`
- Agent 2 → Agent 3: `{results: {source: {found, data, confidence, data_quality}}, failures[], total_latency_ms}`
- Agent 3 → Agent 4: `{findings[], contradictions[], cost_breakdown{}, overall_risk}`

**Merge prompt (pre-write before Hour 1 benchmark):**
If NIM latency >15s/300 tokens, use merged Agent 3+4 prompt: Agent 3 system instructions + "Additionally, synthesize the findings into a plain-English buyer report per the following instructions:" + Agent 4 system instructions. Pre-write both the split and merged variants before starting the benchmark.

**TRCA fallback:** If ArcGIS authentication fails or rate-limits, serve flood zone data from a pre-downloaded GeoJSON for the Toronto bounding box (`data/trca_floodplain_toronto.geojson`). Download this before the hackathon starts.

## Expanded Data Sources (discovered post-plan, documented in DATA_PARAMETERS.md)

The deep research pass on Toronto Open Data (1,052 datasets catalogued) identified 10 additional parameters buildable with the same CKAN API patterns. These are categorized as V1.5 (post-hackathon polish) and documented in full in [`DATA_PARAMETERS.md`](./DATA_PARAMETERS.md).

**V1.5 additions (same CKAN pattern, minimal effort):**
- Heritage Formerly Listed (`heritage-formerly-listed`, ~3,700 records) — Bill 23 delistings
- Residential Fire Inspections (`residential-fire-inspection-results`, 124,414 records) — building-level Ontario Fire Code compliance
- Building Violations (`building-construction-demolition-violations`, 48,035 records) — open compliance folders
- Basement Flooding Study Areas (`basement-flooding-study-areas`, 67 subsewersheds)
- Property Tax Relief Programs (hardcoded by buyer profile)
- Transit Dividend (`ttc-routes-and-schedules`, GTFS) — already in PDF frontend

**V2 additions (require spatial joins or external APIs):**
- Committee of Adjustment Applications (35,984 records)
- Preliminary Zoning Reviews (220,052 records)
- Zoning By-law Restrictions (11,719+ zones)
- TSSA Elevator Violations (external)
- Neighbourhood Crime Rates (158 neighbourhoods)

**Spec alignment (from meridian_spec_final.txt v3):**
- Three-layer signal architecture: observed / inferred / simulated
- Buyer profile input (first-time / investor / downsizer) drives tax rate selection, rebate eligibility
- Multi-residential property tax rate (0.01208792, ~57% higher than residential)
- Property-type-specific assessed value multipliers (condo 0.90, detached suburban 0.55, etc.)
- Maintenance Complexity Signal and Future Tax Pressure Signal as composite inferred signals (always Low confidence)
- RAG corpus for Agent 4 conditional retrieval
- Wording rules: no "red flag," no dollar figures in flood output, no MPAC predictions

Total parameter inventory: **43 parameters + 6 user inputs** across 5 tiers, mapped to **29 CKAN datasets**.
