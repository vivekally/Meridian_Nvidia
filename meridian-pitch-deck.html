How It Works
A four-agent reasoning pipeline.
Agent 01
Geocode
Nominatim resolves the address to lat/lon. Heritage, Flood, and Development sources are routed based on property type.
DETERMINISTIC
Agent 02
Retrieval
Parallel async fan-out: Heritage CKAN (~12k), TRCA ArcGIS flood zones, Development CKAN (~26k). GPU haversine via cuSpatial.
DETERMINISTIC · cuSpatial
Agent 03
Cost Engine
All dollar math: LTT, property tax 10y, CMHC, 2-term mortgage. 10,000 CuPy trajectories produce P10 / P50 / P90 risk bands.
DETERMINISTIC · CuPy
Agent 04
Synthesis
Nemotron-3 Nano 30B narrates the report and negotiation scripts. RAG-grounded via NV-EmbedQA + NV-RerankQA. Never invents a number.

The NVIDIA Stack
Eight NVIDIA technologies. One device. No cloud.
Nemotron-3 Nano 30B via vLLM. NVFP4, Blackwell-native. Narrates reports and powers the chat assistant — but the deterministic engine owns every number. The model never invents a figure.
GPU-accelerated at every stage. CuPy runs 10,000 Monte Carlo trajectories. RAPIDS cuDF + cuSpatial handle haversine proximity across Heritage, Flood, and Development datasets.
Local embeddings, reranking, and speech. NV-EmbedQA (1B) grounds RAG and chat memory. NV-RerankQA (1B) selects the most relevant land-law passages. NeMo ASR (0.6B) transcribes voice input on-device.
